`SDK`，即Software Development Kit



#### 不错`SDK`的原则

我认为一个不错的`SDK`应该包含以下部分：

- 合理的分类
- 简洁，统一风格，自说明的`API`
- 详细的接口说明
- 移植说明
- 调用流程
- 日志，失效分析
- 示例demo
- 自动生成的文档



辅助发布

1. 单元测试
2. 自动测试代码
3. checklist   for release



#### 文档的维护

1. 文档应该是自动生成的。
2. 精简，减少重复，避免冗余
3. 不错的工具： doxygen markdown

#### 示例代码

好的示例代码，可以让然马上开始开发。

1. 开箱即用
2. 符合编程规范；
3. 正确且可拷贝；
